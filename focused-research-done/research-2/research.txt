Error Response Catalog Findings
1. CLOB API Error Codes
The docs provide a detailed Insert Error Messages table for order placement:
Error	Description
INVALID_ORDER_MIN_TICK_SIZE	Price breaks minimum tick size rules
INVALID_ORDER_MIN_SIZE	Size lower than minimum
INVALID_ORDER_DUPLICATED	Same order already placed
INVALID_ORDER_NOT_ENOUGH_BALANCE	Insufficient balance/allowance
INVALID_ORDER_EXPIRATION	Expiration time is in the past
INVALID_ORDER_ERROR	System error inserting order
EXECUTION_ERROR	System error executing trade
ORDER_DELAYED	Order match delayed
FOK_ORDER_NOT_FILLED_ERROR	FOK order couldn't be fully filled
MARKET_NOT_READY	Market not accepting orders yet
Authentication errors include: INVALID_SIGNATURE, NONCE_ALREADY_USED, Invalid Funder Address.
2. Retryable vs Fatal
Retryable: ORDER_DELAYED, MARKET_NOT_READY (success=false but temporary)
Fatal: Most errors with success=true but non-empty errorMsg indicate permanent rejection
3. Rate Limit Response
Rate limits use Cloudflare throttling - requests are delayed/queued rather than rejected. The docs don't specify an explicit error response format for rate limits.
4. Closed Market Orders
The MARKET_NOT_READY error covers markets not accepting orders. Markets have an accepting_orders: boolean field you can check beforehand.
Gap identified: The docs lack explicit HTTP status codes and a comprehensive error response schema.
