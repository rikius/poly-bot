## Phase 0: Wallet Setup (PREREQUISITE)

**Goal:** Set up a funded trading wallet separate from your main wallet.

### Why a Separate Wallet?
- **Security:** Bot has access to private key - limit exposure
- **Operational separation:** Only fund with what you're willing to risk
- **Polymarket requirement:** Need a "signer" wallet for EIP-712 order signing

### Step 0.1: Create New Wallet

**Option A: Using MetaMask (Easiest)**
1. Open MetaMask → Click account icon → "Add account or hardware wallet" → "Add a new account"
2. Name it "Polybot Trading"
3. Click the 3 dots → "Account details" → "Show private key"
4. Enter your password → Copy the private key
5. **SAVE THIS SECURELY** (password manager, NOT in code)

**Option B: Using CLI (More Secure)**
```bash
# Install cast (part of Foundry)
curl -L https://foundry.paradigm.xyz | bash
foundryup

# Generate new wallet
cast wallet new

# Output will show:
# Address: 0x...
# Private key: 0x...
```

### Step 0.2: Fund the Wallet on Polygon

1. **Get MATIC for gas** (small amount, ~$5 worth)
   - Send from your main wallet, OR
   - Use a bridge from Ethereum, OR
   - Buy on exchange and withdraw to Polygon

2. **Get USDC on Polygon** (your trading capital)
   - Option A: Bridge USDC from Ethereum using https://wallet.polygon.technology/
   - Option B: Send USDC from exchange (Coinbase, Binance) directly to Polygon
   - **Start small:** $50-100 for testing

### Step 0.3: Connect to Polymarket

1. Go to https://polymarket.com
2. Connect with your NEW trading wallet
3. **Deposit USDC** into Polymarket (they use a proxy wallet system)
4. Your API credentials are already set up - they work with any connected wallet

### Step 0.4: Verify Setup

You should have:
- [ ] New wallet address (0x...)
- [ ] Private key saved securely (0x... - 64 hex chars)
- [ ] MATIC for gas (~$5)
- [ ] USDC deposited in Polymarket (~$50-100 to start)
- [ ] API credentials (api_key, secret, passphrase) - you already have these

### Environment Variables (for later)
```bash
# .env file (NEVER commit this)
POLYMARKET_API_KEY=your-api-key
POLYMARKET_SECRET=your-secret
POLYMARKET_PASSPHRASE=your-passphrase
PRIVATE_KEY=0x...your-private-key...
WALLET_ADDRESS=0x...your-wallet-address...
```

1. **`src/risk/circuit_breaker.rs`** - Circuit breaker

### Phase 7: Math Arb Strategy (MVP)

**Goal:** First working strategy - mathematical arbitrage.

#### Step 7.1: Implement Edge Calculator

1. **`src/strategy/edge_calculator.rs`** - Dynamic edge calculation
   - `required_edge = fees + slippage + partial_fill_risk + spread_penalty`
   - Estimate slippage from book depth
   - Increase margin for thin books

#### Step 7.2: Implement MathArbStrategy

1. **`src/strategy/arbitrage.rs`** - MathArbStrategy
   - Implements Strategy trait
   - Detect when YES + NO < $1.00 - required_edge
   - Return TWO OrderIntents (YES leg + NO leg)
   - Urgency::Immediate for taker execution

#### Step 7.3: Implement Partial Fill Handling

1. **`src/execution/partial_fill.rs`** - Unwind logic
   - If one leg fills, other doesn't → unwind filled leg
   - Never hold unhedged > 500ms
   - Sell at best bid to close

#### Step 7.4: Validation Checkpoint (PAPER TRADING)
- [ ] Strategy detects arb opportunities
- [ ] Orders placed correctly (paper mode)
- [ ] Partial fill handling works
- [ ] P&L tracking accurate
